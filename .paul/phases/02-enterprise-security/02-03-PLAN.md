<objective>
## Goal
Add a direct file serving endpoint that streams file content through the API with authentication and Cerbos permission checks, eliminating the requirement for publicly accessible S3.

## Purpose
Currently files are served via presigned S3 URLs generated by `AttachmentUrlEnricher`. This requires S3 to be publicly accessible and bypasses API authentication — anyone with the presigned URL can access the file. A direct streaming endpoint adds authentication enforcement, Cerbos permission checks, and works in environments where S3 is on an internal network.

## Output
- `S3StorageService.streamObject()` — streams file content from S3
- `FileController` — `GET /api/files/{storageKey}` endpoint that streams files with auth + permissions
- Content-Type detection and Content-Disposition headers
- Unit tests
</objective>

## Key Architecture Facts
- S3StorageService: presigned URL generation only (no getObject/streaming)
- S3ConfigProperties: endpoint, publicEndpoint, bucket, region, accessKey, secretKey, maxFileSize
- AttachmentUrlEnricher: ResponseBodyAdvice enriching attachment records with downloadUrl
- Files stored with key pattern: tenant-id/collection-id/record-id/uuid/filename.ext
- S3-compatible storage (Garage, MinIO) via AWS SDK v2
- ConditionalOnProperty: kelta.storage.s3.enabled=true
- Gateway routes requests to worker — auth headers (X-User-Email, X-Cerbos-Scope) already present

<acceptance_criteria>

## AC-1: Direct file streaming via API
```gherkin
Given a file exists in S3 storage
When an authenticated user requests GET /api/files/{storageKey}
Then the file content is streamed through the API response
And the response includes correct Content-Type header
And the response includes Content-Disposition header (inline for images/PDF, attachment for others)
And the response includes Content-Length header
```

## AC-2: Authentication required
```gherkin
Given the file serving endpoint
When an unauthenticated request is made
Then a 401 Unauthorized response is returned
And the file is not served
```

## AC-3: Tenant-scoped file access
```gherkin
Given a file belongs to tenant-A (storageKey starts with tenant-A/)
When a user from tenant-B requests the file
Then a 404 response is returned (not 403, to avoid information leakage)
And the file is not served
```

## AC-4: File not found handling
```gherkin
Given a storageKey that does not exist in S3
When the file is requested
Then a 404 response is returned
```

## AC-5: Range request support
```gherkin
Given a large file in S3
When a client sends a Range request header
Then the response includes only the requested byte range
And the response status is 206 Partial Content
And Content-Range header is set correctly
```

## AC-6: S3StorageService streaming capability
```gherkin
Given S3StorageService
When streamObject(storageKey) is called
Then it returns an InputStream of the S3 object content
And the object's content type and content length are available
```

## AC-7: Path traversal prevention <!-- audit-added -->
```gherkin
Given a storageKey containing path traversal sequences (../, %2e%2e/, etc.)
When the file is requested
Then the request is rejected with 400 Bad Request
And the traversal attempt is logged as a security event
```

## AC-8: InputStream resource cleanup <!-- audit-added -->
```gherkin
Given a file is being streamed to the client
When the streaming completes (success or error)
Then the S3 ResponseInputStream is closed
And the underlying HTTP connection is released back to the pool
```

## AC-9: Content-Disposition filename sanitization <!-- audit-added -->
```gherkin
Given a storageKey with a filename containing special characters (newlines, quotes, semicolons)
When Content-Disposition is set
Then the filename is sanitized to prevent header injection
And only alphanumeric, dash, underscore, dot characters are preserved
```

## AC-10: File access audit logging <!-- audit-added -->
```gherkin
Given an authenticated user requests a file
When the file is served (success or denied)
Then a security audit log entry is emitted with: user, storageKey, result, content-length
```
