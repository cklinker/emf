<objective>
## Goal
Enable tenants to use custom domains (CNAME routing) so their users access the platform at `app.acme.com` instead of `platform.kelta.io/acme`.

## Purpose
Custom domains are a core enterprise requirement — white-labeling the platform URL for each tenant. The gateway resolves the tenant from the `Host` header when a custom domain is configured, bypassing the slug-based URL prefix.

## Output
- `CustomDomainFilter` — resolves tenant from Host header for custom domains
- `tenant_custom_domain` table — maps domains to tenants
- Admin API for domain management
- Unit tests
</objective>

<context>
## Source Files
@kelta-gateway/src/main/java/io/kelta/gateway/filter/TenantSlugExtractionFilter.java
@kelta-gateway/src/main/java/io/kelta/gateway/cache/GatewayCacheManager.java

## Key Architecture Facts
- TenantSlugExtractionFilter: extracts tenant from /{slug}/api/... URL path
- GatewayCacheManager: caches tenant slug→id mapping from worker
- Custom domains would resolve tenant from Host header BEFORE slug extraction
- Gateway is reactive (WebFilter chain)
</context>

<acceptance_criteria>

## AC-1: Custom Domain Resolution
```gherkin
Given a tenant "acme" has custom domain "app.acme.com" configured
When a request arrives with Host: app.acme.com
Then the tenant is resolved to "acme" from the domain mapping
And the request proceeds without requiring a slug prefix in the URL
```

## AC-2: Fallback to Slug Resolution
```gherkin
Given a request with Host header that doesn't match any custom domain
When the gateway processes the request
Then it falls back to slug-based tenant resolution (existing behavior)
```

## AC-3: Domain Registration API
```gherkin
Given an admin for tenant "acme"
When they POST /api/admin/domains with {"domain": "app.acme.com"}
Then the domain is registered and associated with their tenant
And the gateway's domain cache is refreshed
```

## AC-4: Domain Uniqueness
```gherkin
Given "app.acme.com" is already registered to tenant "acme"
When another tenant tries to register "app.acme.com"
Then the request is rejected with 409 Conflict
```

## AC-5: Domain Removal
```gherkin
Given a custom domain is registered
When DELETE /api/admin/domains/{domainId} is called
Then the domain mapping is removed
And requests to that domain fall back to slug resolution
```

## AC-6: Host Header Sanitization <!-- audit-added -->
```gherkin
Given a request with a malformed Host header (containing paths, null bytes, or other injection attempts)
When the CustomDomainFilter processes it
Then the Host header is sanitized before cache lookup
And only valid domain names are accepted
```

## AC-7: Platform Domain Protection <!-- audit-added -->
```gherkin
Given a tenant attempts to register a platform-owned domain (e.g., platform.kelta.io, *.kelta.io)
When the registration is processed
Then it is rejected with 400 "Reserved domain"
And a configurable blocklist of reserved domain patterns is maintained
```

## AC-8: Domain Cache
```gherkin
Given custom domains are configured
When the gateway starts or receives a cache refresh event
Then all custom domain mappings are loaded into an in-memory cache
And domain lookups are O(1) (no database query per request)
```

    1. Create V110 migration:
       ```sql
       CREATE TABLE tenant_custom_domain (
           id        VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid()::text,
           tenant_id VARCHAR(36) NOT NULL REFERENCES tenant(id),
           domain    VARCHAR(255) NOT NULL UNIQUE,
           verified  BOOLEAN NOT NULL DEFAULT false,
           created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
       );
       CREATE INDEX idx_custom_domain_domain ON tenant_custom_domain(domain);
       ```

    2. Create CustomDomainFilter (WebFilter, order BEFORE TenantSlugExtractionFilter):
       - Extract Host header from request
       - Strip port if present (host:port → host)
       - Look up domain in cache (Map<String, String> domain→tenantSlug)
       - If found: set tenant attributes on exchange, skip slug extraction
       - If not found: pass through (TenantSlugExtractionFilter handles)
       - Cache refreshed from worker at startup + on Kafka config event

    3. Extend GatewayCacheManager:
       - Add loadCustomDomains(): fetches domain→slug mappings from worker
       - Add resolveDomain(host): returns Optional<tenantSlug>
       - Refresh on startup and on config change events

    4. Create TenantDomainController (@RestController, /api/admin/domains):
       - GET / — list domains for tenant
       - POST / — register domain (check uniqueness, tenant-scoped, reject reserved domains) <!-- audit-added -->
       - DELETE /{id} — remove domain
       - Publish config change event on register/delete (for cache refresh)

    5. Tests:
       - CustomDomainFilterTest: domain resolves tenant, unknown domain falls through, port stripped
       - TenantDomainControllerTest: register, list, delete, uniqueness conflict

    Avoid: DNS verification in this plan (verified flag exists for future use).
    Avoid: TLS certificate management (handled by ingress/reverse proxy).
